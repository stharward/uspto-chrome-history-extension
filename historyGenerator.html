Instructions

The Google Search History Generator is a tool used for selecting and saving search queries as a PDF file, to be imported into an OC Set. A full QRG on installation and use of this tool can be found on the PE2E OC Training and Reference Page.

Creating the “Printable” Search History

  1. In the list above, click in the Show entries from past field and select an appropriate timeframe from the expanded menu.
  2. Click the checkboxes next to search strings that should appear on the resulting search history.
  3. Click the Generate Printable Search History button. A Print window displays. Click the Destination field, select Save as PDF, and click the Save button.
  4. If you need to generate multiple separate search history listings (e.g. you have searches for two applications mixed together), generate one, then reload this page, and generate the second one.

Putting the Search History in e-Red Folder and OC action

  1. In OC, click the Import File(s) button in the toolbar. An Import Files window opens.
  2. If the Search History is saved in an e-Red Folder for the application, the file will be listed in the Import Files window; or navigate to the location of the saved search history file. Drag the PDF file to the “Drag and drop files here” portion of the Import File(s) window. The file will then be listed.
  3. Click to check the selection box next to the Search History file in the list.
  4. If not automatically populated, click the Doc Code field and select SRNT from the expanded list.
  5. Click the Import button and verify the file appears in the document list and was properly imported.

Getting Further Help

Users can direct questions about PE2E-OC or this extension to the ITRP Team. For any problems importing the Search History into an OC set, please submit a PASM ticket to the PE2E Official Correspondence (OC) team.

Version History and ChangeLog

Version 5.3 (22 Jul 2020)

  • Reinstated handler for old Google Patents search interface.
  • Updated documentation for OC.

Version 5.2 (16 Apr 2020)

  • This extension now has official OCIO approval!
  • Support Google Web searches.
  • Removed support for old, discontinued Google Patents search interface.
  • Updated documentation for OC.

Version 5.1 (18 Oct 2017)

  • Query processor update to handle Google tweaking the format of the Patents search URL.

Version 5 (27 Sep 2017)

  • Google launched a new version of Scholar, necessitating a rewrite of the query parameter extraction function.

Version 4 (1 Mar 2017)

  • When eliminating duplicates of a query, keep the first instance of the query, rather than the last.

Version 3 (28 Feb 2017)

  • Added a function to filter out queries that were executed, but weren’t displayed long enough to be reviewed. For example, if you realized that you misspelled a search term, and immediately ran a new query with the new term, you can filter out the old query. This is particularly useful when building a complicated query in Google Patents, because Chrome runs a new query each time you enter a new search term, and you probably don’t care about keeping all the intermediate queries that were run while you were building up the whole query.
  • Parse more fields from the Google Patents search.
  • Format the output of Google Patents searches a little differently to make it easier to understand the structure of queries that used proximity or boolean operators.
  • Ignore spurious entries in the Google Patents list caused when the URL of a publication also includes the query that yielded the publication.

Version 2 (1 Oct 2015)

  • Updated to handle the July 2015 overhaul of the Google Patents interface. The old interface is still around, so this extension still handles it for now.

Version 1 (29 May 2015)

  • Initial release. Supports Google Books, Images, Patents and Scholar.
